# 2026-02-18 갤러리 모달 슬라이드 애니메이션 추가

## 작업 개요
갤러리 모달에서 이미지를 슬라이드할 때 부드러운 애니메이션 효과를 추가했습니다.

## 변경 사항

### 1. GallerySection.jsx
- 모달 애니메이션을 위한 state 추가:
  - `prevModalImage`: 이전 이미지 인덱스 저장
  - `isModalTransitioning`: 애니메이션 진행 중 여부
  - `modalDirection`: 슬라이드 방향 (1: 오른쪽, -1: 왼쪽)
- `navigateImage` 함수 수정:
  - 애니메이션 중에는 중복 호출 방지
  - 이전 이미지와 방향을 저장하여 애니메이션 상태 관리
  - 600ms 후 애니메이션 상태 초기화
- `openModal` 함수 수정:
  - 모달 열 때 애니메이션 상태 초기화
- 모달 이미지 렌더링 로직 수정:
  - 이전 이미지와 현재 이미지를 동시에 렌더링하여 슬라이드 효과 구현
  - 방향에 따라 다른 애니메이션 클래스 적용

### 2. GallerySection.css
- 모달 이미지 애니메이션 스타일 추가:
  - `.modal-image-wrapper`: 상대 위치 지정 및 중앙 정렬
  - `.modal-image-in-right`: 오른쪽에서 들어오는 애니메이션
  - `.modal-image-in-left`: 왼쪽에서 들어오는 애니메이션
  - `.modal-image-out.slide-out-left`: 왼쪽으로 나가는 애니메이션
  - `.modal-image-out.slide-out-right`: 오른쪽으로 나가는 애니메이션
- 키프레임 애니메이션 정의:
  - `modalSlideInFromRight`: 오른쪽에서 슬라이드 인
  - `modalSlideInFromLeft`: 왼쪽에서 슬라이드 인
  - `modalSlideOutToLeft`: 왼쪽으로 슬라이드 아웃
  - `modalSlideOutToRight`: 오른쪽으로 슬라이드 아웃

## 동작 방식
1. 사용자가 모달에서 스와이프하거나 키보드로 이미지를 변경할 때
2. 이전 이미지는 슬라이드 아웃 애니메이션과 함께 사라지고
3. 새 이미지는 슬라이드 방향에 맞게 슬라이드 인 애니메이션과 함께 나타남
4. 애니메이션 지속 시간: 600ms (ease-out)

## 기술적 세부사항
- 애니메이션 중 중복 호출 방지를 위해 `isModalTransitioning` 상태 사용
- 스와이프 방향에 따라 다른 애니메이션 적용 (자연스러운 UX)
- 이전 이미지는 `position: absolute`로 현재 이미지 위에 오버레이
- 애니메이션 완료 후 상태 정리로 메모리 누수 방지
