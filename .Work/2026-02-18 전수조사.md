## 1. 사용하지 않는 코드 (삭제 또는 복구 검토)

### 1.1 라우트/렌더링에서 제외된 컴포넌트

| 대상 | 위치 | 상황 |
|------|------|------|
| **Navigation** | `App.jsx` | `import`는 있으나 JSX에서 `{/* <Navigation /> */}` 로 주석 처리되어 화면에 안 나옴. |
| **celebrationSection** | `src/components/celebration/celebrationSection.jsx` | `App.jsx`에 import·렌더링 없음. 완전 미사용. |

→ **제안**:  
- Navigation: 사용할 계획이 있으면 주석 해제, 없으면 import 포함 제거.  
- celebrationSection: 사용할 계획 없으면 파일 삭제 검토.

---

### 1.2 어디에서도 import되지 않는 컴포넌트/파일

| 대상 | 위치 | 비고 |
|------|------|------|
| **ContactParents** | `src/components/main/components/ContactParents.jsx` + `ContactParents.css` | MainSection이 연락하기를 인라인으로 직접 구현하고 있어, ContactParents는 호출처 없음. |
| **MessageCard** | `src/components/cards/MessageCard.jsx` + `MessageCard.css` | MessageBoard는 `<div className="message-card">` 로 직접 마크업. MessageCard 컴포넌트 미사용. |
| **AccountCard** | `src/components/cards/AccountCard.jsx` + `AccountCard.css` | AccountSection은 자체 마크업(`.account-item` 등). AccountCard는 **celebrationSection에서만** 사용되는데, celebrationSection 자체가 미사용. |
| **Button / ContactButton** | `src/components/buttons/` (Button.jsx, ContactButton.jsx, 각 CSS, index.js) | 사용처: ContactParents, AccountCard, celebrationSection 뿐. 위 셋이 모두 미사용이므로 **현재 앱 트리에서는 버튼 컴포넌트 사용처 없음**. |

→ **제안**:  
- ContactParents: 삭제 또는, 나중에 MainSection을 ContactParents 기반으로 리팩터할 계획이 있으면 그때까지 보류.  
- MessageCard: MessageBoard에서 `<MessageCard />` 로 교체해 쓰거나, 쓸 계획 없으면 MessageCard(및 관련 CSS) 삭제 검토.  
- AccountCard / Button / ContactButton: celebrationSection·ContactParents 정리 방향(삭제 vs 복구)에 맞춰 함께 정리.

### 1.3 스타일 파일 중 import되지 않는 것

| 파일 | 비고 |
|------|------|

→ **제안**: 사용할 계획 없으면 삭제; 필요하면 globals 등에 통합 후 단일 진입점만 유지.

## 2. 중복 코드 (공통화 후보)

### 2.1 하단 토글 패널 (연락하기 / 메시지 작성)

- **MainSection**: `contact-toggle-panel` + body overflow + 외부 클릭 닫기 + `querySelector('.contact-toggle-panel')`  
- **MessageBoard**: `message-toggle-panel` + body overflow + 외부 클릭 닫기 + `querySelector('.message-toggle-panel')`  

로직·구조가 거의 동일함.

- **CSS**:  
  - `.contact-toggle-panel` / `.message-toggle-panel`  
  - `.contact-toggle-header` / `.message-toggle-header`  
  - `.contact-toggle-close` / `.message-toggle-close`  
  - `.contact-toggle-title` / `.message-toggle-title`  
  → 클래스명만 다르고 스타일 동일.  
- **JS**:  
  - `open` 시 `document.body.style.overflow = 'hidden'`, 닫을 때 `unset`  
  - `setTimeout` 후 document에 click 리스너 등록, 패널 밖 클릭 시 닫기  

→ **제안**:  
- 공통 컴포넌트 1개 도입 (예: `BottomSheet` 또는 `BottomPanel`): `open`, `onClose`, `title`, `children`.  
- 훅 1개 (예: `useBottomPanel`): open 상태, body overflow, 외부 클릭(닫기) 처리.  
- 공통 CSS 클래스 1세트 (예: `.bottom-sheet-panel`, `.bottom-sheet-header`, `.bottom-sheet-close`).  
- MainSection·MessageBoard는 위 컴포넌트/훅만 사용하도록 변경.  
→ 한 곳만 수정해도 두 패널이 동일하게 동작해, “한 곳 고치면 다른 데 깨짐” 가능성을 줄일 수 있음.

---

### 2.2 전화번호 포맷 (tel:/sms: 링크용)

- **MainSection.jsx**: `formatPhoneForLink(phone) => phone.replace(/-/g, '')`  
- **ContactParents.jsx**: 동일 함수 로컬 정의  

→ **제안**:  
- `src/utils/formatPhone.js` 등에 `formatPhoneForLink` 한 번만 정의.  
- MainSection(및 나중에 ContactParents를 다시 쓸 경우)에서 import해서 사용.

---

### 2.3 모달/패널 공통 패턴 (body overflow)

- RSVPModal, MainSection(연락 패널), MessageBoard(메시지 패널), ContactParents(미사용):  
  모두 “열림 시 `body overflow hidden`, 닫힘/언마운트 시 `unset`” 동일 패턴.

→ **제안**:  
- 훅 `useLockBodyScroll(open)` 같은 형태로 한 곳에 두고, RSVPModal·BottomSheet(연락/메시지)에서 재사용.  
- 외부 클릭 닫기는 `useBottomPanel`에 포함시키면, “열림 + 스크롤 잠금 + 닫기”가 한 패턴으로 정리됨.

## 3. 공통으로 뺄 수 있는 부분

### 3.1 이미 공통인 것 (유지)

- `SectionTitle`: 여러 섹션에서 사용 중.  
- `globals.css` + `typography.css`: 변수·타이포·`.container` 등.  
- `ToastContext` / `useToastContext`: Toast 사용처에서 일관 사용.  
- `scrollAnimation.js`: 섹션/푸터 공통 스크롤 애니메이션.  
- `wedding.js`: COUPLE, VENUE, ACCOUNTS 등 단일 소스.

### 3.2 공통 컴포넌트/훅으로 빼면 좋은 것

| 항목 | 내용 |
|------|------|
| **BottomSheet(이름 가칭)** | 연락하기 패널 + 메시지 작성 패널을 하나의 재사용 컴포넌트로. (위 2.1) |
| **useLockBodyScroll** | 모달/패널 열림 시 body 스크롤 잠금. (위 2.3) |
| **useBottomPanel** | open 상태 + 스크롤 잠금 + 외부 클릭 닫기. (위 2.1, 2.3) |
| **formatPhoneForLink** | `src/utils/` 한 곳에 두고 import. (위 2.2) |

## 4. 기타 정리

### 4.1 CSS 진입점

- `main.jsx`: `import './index.css'`  
- `App.jsx`: `import './styles/globals.css'`  
- `index.css`: `#root`, `.App` 등 레이아웃만 정의하고, 주석에 “globals.css로 대체됨”이라고 되어 있음.

→ **제안**:  
- 실제로 전역 스타일이 globals에 몰려 있다면, `index.css` 내용을 globals에 합치고 `main.jsx`에서는 `globals.css`만 import 하거나,  
- 역할을 명확히 나누려면 (예: index = #root/App만, globals = 변수·리셋·공통) 주석/파일명을 그에 맞게 정리.  
- 한 곳만 수정해도 레이아웃이 바뀌지 않도록, “전역 레이아웃은 어디에서 정의하는가”를 한 군데로 통일하는 것이 좋음.

### 4.2 assets 경로

- Footer: `../assets/images/rose-flower.png`  
- RSVPModal: `../assets/images/flowers.png`  
- MapSection: `../../assets/images/map.png`  
- GallerySection: `../../assets/images/flower-rose.png`  

(프로젝트 내에 `assets` 폴더가 없거나 다른 위치에 있을 수 있음. 빌드/이미지 경로 오류 시 여기부터 확인 권장.)

## 5. 권장 진행 순서 (수정 전 논의용)

1. **미사용 코드 정리 방향 확정**  
   - Navigation: 사용 여부.  
   - celebrationSection: 삭제 vs 복구.  
   - ContactParents, MessageCard, AccountCard, Button/ContactButton: 삭제 vs 도입(리팩터) 결정.  
   - colors.css, fonts.css: 삭제 vs 통합.

2. **공통화 순서**  
   - 전화번호: `formatPhoneForLink` 유틸 먼저 추출.  
   - 그다음 `useLockBodyScroll` → `useBottomPanel` + `BottomSheet` 로 연락/메시지 패널 통합.  
   - 이후 필요 시 RSVPModal에도 `useLockBodyScroll` 적용.

3. **CSS/엔트리 정리**  
   - index.css vs globals.css 역할 정리 및 import 한 곳으로 통일.

4. **소소한 수정**  
   - MainSection에서 VENUE import 제거.

이 순서로 하면 “한 군데만 고치면 다른 데 깨지는” 상황을 줄이면서, 삭제·공통화를 단계적으로 적용할 수 있습니다.  
실제 수정은 위 방향을 사용자와 맞춘 뒤 진행하는 것을 권장합니다.
