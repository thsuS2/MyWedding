## 🎯 추천: Supabase

### 이유
1. **이미 설정되어 있음** - 추가 작업 최소화
2. **실시간 확인 가능** - 관리자가 즉시 확인 가능
3. **무료 티어 충분** - RSVP 데이터는 많지 않을 것으로 예상
4. **일관성** - 기존 메시지 보드와 동일한 백엔드 사용

### 예상 데이터량
- 참석자 수: 약 100-200명 가정
- 레코드당 크기: 약 200 bytes
- 총 데이터: 약 20-40 KB (매우 작음)
- Supabase 무료 티어: 500MB (충분함)

## 📊 테이블 구조 (Supabase)

```sql
CREATE TABLE rsvp (
  id BIGSERIAL PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  side TEXT NOT NULL, -- '신랑측' or '신부측'
  name TEXT NOT NULL,
  companion TEXT, -- 동행인 (선택)
  meal TEXT NOT NULL, -- '예정', '안함', '미정'
  message TEXT -- 추가 메시지 (선택)
);

-- RLS 정책
CREATE POLICY "Enable insert access for all users" 
ON rsvp FOR INSERT 
WITH CHECK (true);

CREATE POLICY "Enable read access for all users" 
ON rsvp FOR SELECT 
USING (true);
```
